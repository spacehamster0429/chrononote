# Security Policy

## Supported version

ChronoNote는 `main` 브랜치와 [운영 중인 최신 버전](https://note.serika.duckdns.org)을 기준으로 보안 수정이 제공됩니다.

## Reporting a vulnerability

취약점, 재현 절차, 영향을 받는 기능과 가능한 완화 방법을 GitHub의 비공개 취약점 신고 기능으로 보내주세요.

비공개 신고 기능을 사용할 수 없다면 [GitHub Issues](https://github.com/spacehamster0429/chrononote/issues)에 취약점 세부 내용이나 계정 정보를 적지 말고, 비공개 연락이 필요하다는 사실만 남겨주세요.

공개 이슈에는 다음 정보를 올리지 마세요.

- 비밀번호, 세션 쿠키, OAuth 코드 또는 Personal Access Token
- 이메일, 사용자 ID, 메모 내용과 첨부파일
- 아직 수정되지 않은 취약점의 구체적인 악용 절차
