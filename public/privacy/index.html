개인정보 처리방침

버전 2026-07-06 · 시행일 및 최종 수정: 2026년 7월 6일

1. 요약

ChronoNote는 계정과 개인 작업공간을 제공하기 위해 필요한 정보만 처리합니다. 광고, 이용자 프로파일링, 데이터 판매 또는 외부 분석 도구를 사용하지 않습니다.

핵심: 비로그인 체험 메모는 현재 브라우저에만 저장됩니다. 로그인 메모와 첨부파일은 운영자가 관리하는 서버에 저장되므로 운영자는 시스템 관리상 접근할 수 있습니다.

2. 처리하는 개인정보

이메일 회원가입 시 필수

이메일 주소, 솔트가 적용된 비밀번호 해시, 이용약관·개인정보 동의 시각과 문서 버전, 만 14세 이상 확인 시각

Google 회원가입·로그인 시

Google 계정 식별자와 이메일 주소, 동의 기록. Google 비밀번호는 ChronoNote가 받거나 저장하지 않습니다.

서비스 이용 중 생성·저장

메모 제목·본문·태그·폴더, 첨부파일, 자동 복구본, Git 변경 기록, 검색 인덱스, 테마·자동 저장 등 계정 설정

GitHub 동기화 선택 시

이용자가 지정한 저장소 이름과 암호화된 Personal Access Token(PAT), Git 동기화 결과

기술 정보

세션 쿠키, 요청 IP 기반의 메모리 내 속도 제한 정보, 요청 시각과 오류 내용이 포함될 수 있는 운영 로그

메모에는 이용자가 자유롭게 작성한 정보가 포함될 수 있습니다. 주민등록번호, 금융정보, 건강정보 등 불필요한 민감정보나 타인의 개인정보를 저장하지 마세요.

3. 처리 목적과 보유 기간

  • 계정·인증: 가입, 로그인, 재접속, 계정 보호 및 악용 방지. 회원탈퇴 요청 후 48시간의 복구 기간까지 보관한 뒤 삭제합니다.
  • 메모 서비스: 작성·검색·복구·Git 히스토리·첨부·내보내기 제공. 계정 유지 기간과 탈퇴 복구 기간 동안 보관합니다.
  • GitHub 동기화: 이용자가 요청한 저장소 백업. 설정에서 토큰을 교체하거나 계정을 삭제할 때까지 보관합니다.
  • 세션: 로그인 유지. 세션 쿠키는 발급 후 최대 30일이며 로그아웃하면 서버에서 해당 세션을 무효화합니다.
  • 보안·오류 대응: 속도 제한 값은 서버 메모리에서 짧은 제한 시간 동안만 처리됩니다. 오류 로그는 시스템 순환 정책에 따라 필요한 기간만 보관합니다.

다른 법령에 따른 보존 의무가 생기는 경우에는 해당 근거와 기간에 따라 별도 보관할 수 있습니다. 현재 유료 결제·전자상거래 기록은 처리하지 않습니다.

4. 수집·이용 동의

회원가입 시 계정과 작업공간 제공을 목적으로 이메일 또는 Google 계정 식별정보와 동의 기록을 필수로 수집합니다. 동의를 거부할 수 있으나, 필수 계정 정보 처리에 동의하지 않으면 회원가입과 서버 저장 기능을 이용할 수 없습니다. 비로그인 체험 모드는 이용할 수 있습니다.

5. 제3자 제공과 처리 위탁

ChronoNote는 개인정보를 판매하거나 광고 사업자·데이터 브로커에 제공하지 않습니다. 법령에 따른 적법한 요청, 이용자의 별도 요청 또는 서비스·이용자의 안전을 보호해야 하는 경우를 제외하고 임의로 공개하지 않습니다. 서버는 운영자가 직접 관리하며 현재 별도의 메모 저장 처리 수탁업체를 사용하지 않습니다.

6. 외부 서비스와 국외 처리

Google OAuth

이용자가 Google 가입·로그인을 선택하면 이전받는 자 Google LLC(이전 국가: 미국)의 인증 화면과 API를 사용합니다. 인증을 시도한 때 OAuth 코드가 암호화된 통신으로 전송되고, ChronoNote는 Google 계정 식별자와 이메일을 전달받습니다. Google의 보유 기간과 처리 방식은 Google 개인정보처리방침을 따릅니다. Google 방식을 원하지 않으면 이메일 가입을 이용할 수 있습니다.

GitHub 동기화

이용자가 기능을 직접 설정하고 실행한 경우에만 이전받는 자 GitHub, Inc.(이전 국가: 미국)의 인프라로 메모 Git 기록과 저장소 인증정보가 암호화된 통신을 통해 전송됩니다. 전송 시점은 동기화 버튼을 누른 때이며, GitHub 저장소의 보유 기간과 공개 범위는 이용자가 관리하고 GitHub 개인정보처리방침이 적용됩니다. 동기화를 설정하지 않으면 전송되지 않습니다.

7. 쿠키와 브라우저 저장소

로그인 유지와 요청 위조 방지를 위해 chrononote_session HttpOnly 쿠키를 사용합니다. 쿠키는 Secure·SameSite=Lax로 설정되며 최대 30일간 유지됩니다. 비로그인 체험 메모, UI 배치와 테마 설정은 브라우저의 로컬 저장소에 보관될 수 있습니다. 브라우저 설정에서 삭제할 수 있지만 체험 메모도 함께 사라질 수 있습니다.

8. 파기 절차와 방법

회원탈퇴를 요청하면 계정에 탈퇴 시각을 기록하고 로그아웃 처리합니다. 48시간 안에 다시 로그인하면 복구할 수 있으며, 이후에는 계정 DB 레코드와 작업공간의 메모·첨부·자동저장·Git 기록·검색 DB를 파일시스템과 SQLite에서 삭제합니다. 배포 전 안전을 위해 생성한 백업 사본은 접근을 제한하고 최대 30일 안에 자동 삭제합니다.

GitHub에 동기화한 사본과 이용자가 내려받은 ZIP은 ChronoNote가 직접 삭제할 수 없으므로 이용자가 해당 위치에서 삭제해야 합니다.

9. 안전성 확보 조치

  • HTTPS 전송, HttpOnly·Secure 세션 쿠키와 CSRF 검증
  • 비밀번호의 salted scrypt 해시 저장과 GitHub PAT의 AES-256-GCM 암호화
  • 사용자별 작업공간 격리, 파일·DB 접근 권한 제한
  • 비루트·읽기 전용 컨테이너, 최소 권한과 요청 속도·용량 제한
  • 로그아웃 세션 폐기, Google 계정 자동 병합 차단과 서버 측 명령 검증

다만 어떠한 전송·저장 방식도 절대적인 안전을 보장할 수는 없습니다.

10. 이용자의 권리

이용자는 설정에서 자신의 데이터를 수정·내보내기하고 회원탈퇴를 요청할 수 있습니다. Google 전용 계정은 먼저 비밀번호를 등록한 뒤 앱에서 탈퇴할 수 있습니다. 그 밖의 열람·정정·삭제·처리정지 요청은 아래 문의 채널을 이용해 주세요. 본인 확인을 위해 계정 이메일 등 최소한의 정보가 필요할 수 있습니다.

11. 만 14세 미만 아동

ChronoNote 회원가입은 만 14세 이상만을 대상으로 하며 법정대리인 동의 확인 기능을 제공하지 않습니다. 만 14세 미만 이용자는 계정을 만들지 마세요. 만 14세 미만 아동의 계정이 확인되면 본인 또는 법정대리인의 요청에 따라 확인 후 삭제합니다.

12. 방침 변경

처리 항목이나 기능이 바뀌면 이 문서와 상단의 버전·최종 수정일을 갱신합니다. 이용자 권리에 중요한 변경은 서비스 안에서 알리거나 필요한 경우 다시 동의를 받습니다.
